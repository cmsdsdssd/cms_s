# CMS 업무 흐름(Workflow) 중심 재설계안

## 🎯 핵심 문제 재정의

**"전체적으로 통일감이 안생긴다"** = 각 메뉴가 따로 놀아서 **업무의 연속성**이 끊긴다

### 현재의 단절 지점 (Breakpoint Analysis)

```
┌─────────────┐    ❌    ┌─────────────┐    ❌    ┌─────────────┐
│  주문 입력   │─────────▶│  주문 관리   │─────────▶│  출고 입력   │
│  /orders    │  페이지   │ /orders_main│  페이지   │ /shipments  │
│             │  이동    │             │  이동    │             │
└─────────────┘         └─────────────┘         └─────────────┘
       ▲                                              │
       │              ❌ 페이지 이동                    │
       │              컨텍스트 상실                    ▼
       │                                         ┌─────────────┐
       │              ❌ 거래처 재선택              │   미수금    │
       └─────────────────────────────────────────▶│    /ar     │
                                                  └─────────────┘

❌ 문제점 요약:
1. 주문 입력 후 관리하려면 페이지 이동
2. 주문→출고는 또 다른 페이지
3. 출고→미수는 또 다른 페이지  
4. 각 페이지마다 거래처 다시 검색
5. 이전 작업 맥락 완전 상실
```

### 뷰 2: 실시간 업무 확장 패널 (Inline Expansion)

**핵심**: 페이지 이동 없이 같은 화면에서 업무 진행

```typescript
// 주문 라인 → 출고 처리 (확장)
function OrderLineWithShipment({ order }) {
  const [expanded, setExpanded] = useState(false);
  
  return (
    <div className="order-line">
      <div className="summary" onClick={() => setExpanded(!expanded)}>
        📝 주문 #001  BRACELET-001 × 10개
        <button onClick={(e) => { e.stopPropagation(); setExpanded(true); }}>
          [🚚 출고처리]
        </button>
      </div>
      
      {expanded && (
        <div className="expansion-panel">
          {/* 출고 입력 폼 - 같은 페이지! */}
          <ShipmentForm 
            orderLineId={order.id}
            onComplete={() => {
              // 완료 후 타임라인 자동 새로고침
              refreshTimeline();
              setExpanded(false);
            }}
          />
        </div>
      )}
    </div>
  );
}
```

**UI 흐름**:
```
1. 주문 라인 클릭
   ↓
2. 확장 패널 열림 (슬라이드다운 애니메이션)
   ↓
3. 출고 정보 입력 (중량, 차감, 공임)
   ↓
4. [출고확정] 클릭
   ↓
5. ✓ 패널 닫히고 타임라인 업데이트
   ↓
6. 같은 화면에서 출고 완료 상태 확인 + [수금등록] 버튼 활성화
```

## 🔄 데이터 흐름: 상태 관리 아키텍처

### 전역 상태 설계 (Zustand / Context)

```typescript
// 업무 컨텍스트 (모든 컴포넌트 공유)
interface WorkContext {
  // 현재 선택된 거래처 (모든 페이지 공유)
  currentParty: Party | null;
  
  // 타임라인 데이터 (캐싱)
  timeline: {
    data: TimelineItem[];
    lastUpdated: Date;
    isLoading: boolean;
  };
  
  // 활성 작업 (확장 패널 상태)
  activeWorks: {
    [workId: string]: {
      type: 'order' | 'shipment' | 'payment';
      expanded: boolean;
      draft: any;  // 임시 저장 데이터
    }
  };
  
  // 알림 및 할 일
  notifications: Notification[];
}

// 작업 흐름 액션
interface WorkActions {
  // 거래처 선택 (글로벌)
  selectParty: (party: Party) => void;
  
  // 타임라인 새로고침
  refreshTimeline: () => Promise<void>;
  
  // 작업 확장/축소
  expandWork: (workId: string, type: string, data: any) => void;
  collapseWork: (workId: string) => void;
  
  // 임시 저장
  saveDraft: (workId: string, draft: any) => void;
  loadDraft: (workId: string) => any;
  
  // 작업 완료 후 다음 작업 연결
  completeAndNext: (completedWork: string, nextWork: string) => void;
}
```

### 페이지 전환 없이 데이터 흐름

```typescript
// 예: 주문 입력 → 출고 처리 → 수금 등록
function handleOrderComplete(order: Order) {
  // 1. 주문 저장
  await saveOrder(order);
  
  // 2. 타임라인에 추가 (실시간)
  addToTimeline({
    type: 'order',
    id: order.id,
    title: `주문 #${order.id}`,
    status: 'pending',
    expanded: false
  });
  
  // 3. 출고 가능 알림 추가
  addNotification({
    type: 'action_required',
    message: '출고 가능한 주문이 있습니다',
    action: () => expandWork(order.id, 'shipment', order)
  });
}

function handleShipmentComplete(shipment: Shipment) {
  // 1. 출고 저장
  await saveShipment(shipment);
  
  // 2. 주문 상태 업데이트 (타임라인에서)
  updateTimelineItem(shipment.orderId, { 
    status: 'shipped',
    shipmentId: shipment.id 
  });
  
  // 3. 수금 등록 버튼 활성화 (같은 화면에서)
  updateTimelineItem(shipment.id, {
    actions: ['payment']  // 수금 버튼 표시
  });
  
  // 4. 재고 자동 차감 (애니메이션으로 표시)
  showInventoryDeduction(shipment.items);
}
```

### 3. 스마트 네비게이션

```typescript
// URL 구조 변경
// 현재: /orders, /shipments, /ar, /inventory
// 개선: /workbench/:partyId?view=orders|shipments|timeline|inventory

function WorkbenchPage() {
  const { partyId } = useParams();
```

## 🎯 핵심 설계 원칙 요약

### 1. **Context Continuity** (컨텍스트 연속성)
- 거래처 선택은 글로벌 상태
- 페이지 이동 없이 뷰 전환
- 이전 작업의 맥락 유지

### 2. **Progressive Disclosure** (점진적 정보 공개)
- 기본적으로 간단한 요약만 표시
- 클릭/확장 시 상세 정보 노출
- 한 화면에 3단계 이상 확장 가능

### 3. **Visual Flow** (시각적 흐름)
- 색상으로 업무 단계 구분
- 애니메이션으로 상태 변화 강조
- 연결선/화살표로 진행 방향 표시

### 4. **Actionable Notifications** (실행 가능한 알림)
- 단순 정보가 아닌 액션 포함
- 클릭 한 번으로 해당 작업 시작
- 우선순위 기반 정렬

- **신규 직원 교육**: 직관적 흐름으로 교육 시간 60% 단축

- **확장성**: 새로운 업무 타입 쉽게 추가 가능

**"데이터는 이미 연결되어 있지만, UI/UX도 같은 맥락으로 연결하라"**

- 데이터베이스 관계: Party → Order → Shipment → AR (연결됨 ✓)
- 현재 UI: Party 선택 → Order 페이지 → Shipment 페이지 → AR 페이지 (단절됨 ✗)
- 개선 UI: Party 선택 → 같은 페이지에서 Order+Shipment+AR (연결됨 ✓)
