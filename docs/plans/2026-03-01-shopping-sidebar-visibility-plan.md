# 쇼핑몰 정보 가시성 개선 실행 계획

## 1) 목표
- 쇼핑몰 라우트에서는 좌측 네비게이터가 쇼핑몰 전용 정보만 보여서 집중도를 높인다.
- 홈 라우트에서는 쇼핑몰 그룹을 완전히 제거해 업무 메뉴의 노이즈를 줄인다.
- 상단 바에서 현재 모드(홈/쇼핑몰/분석)를 항상 표시해 현재 위치를 즉시 인지할 수 있게 한다.
- 쇼핑몰 화면 안에서는 옵션/정책/동기화 상태를 한눈에 파악할 수 있도록 정보 구조를 재구성한다.

## 2) 현재 상태 요약
- 모드 분리:
  - `home`: 일반 업무 라우트
  - `shopping`: `/settings/shopping` 계열
  - `analysis`: `/analysis` 계열
- 좌측 네비:
  - 홈에서는 쇼핑몰 그룹 미노출
  - 쇼핑몰에서는 쇼핑몰 그룹만 노출
  - 분석에서는 분석 그룹만 노출
- 상단 모드 칩:
  - 홈/쇼핑몰/분석 모두에서 `{모드명} 모드` 표시

## 3) 쇼핑몰 화면 정보 구조(IA) 개선안

### A. 3-레이어 구조로 단순화
1. 상단 요약 레이어(현재 상태)
   - 채널 연결 상태
   - 최근 동기화 결과(성공/실패/대기)
   - 오늘 반영 건수/오류 건수
2. 작업 레이어(지금 해야 할 일)
   - 우선 액션 카드 3개 이내(예: 미매핑 처리, 실패 재시도, 정책 충돌 확인)
3. 상세 설정 레이어(관리)
   - 채널 설정, 상품 매핑, 가격 대시보드, 정책/팩터, 동기화 로그

### B. 메뉴 그룹 재편 제안
- `운영 현황`
  - 쇼핑몰 홈
  - 가격 대시보드
  - 동기화 로그
- `설정/연결`
  - 채널 설정
  - 정책/팩터
- `카탈로그/매핑`
  - 상품 매핑

### C. 시각 우선순위 규칙
- 1순위(즉시 대응 필요): 경고/실패/충돌은 강조 카드 + 아이콘 + 색상 태그
- 2순위(당일 작업): 미완료 작업은 체크리스트 블록
- 3순위(참고 정보): 부가 지표는 축약 카드로 배치

## 4) UI 컴포넌트 레벨 실행 항목

### Task 1. 쇼핑몰 홈 카드 재배치
- 파일 후보: `web/src/app/(app)/settings/shopping/page.tsx`
- 작업:
  - 상단 KPI 3개 고정
  - 오류/경고 패널을 상단 우측에 배치
  - 액션 버튼(재동기화, 미매핑 보기, 정책 점검) 우선 노출

### Task 2. 네비게이터 그룹명/순서 개선
- 파일 후보: `web/src/components/layout/nav-items.ts`
- 작업:
  - 쇼핑몰 메뉴 라벨을 업무 순서 중심으로 정렬
  - 그룹명은 사용자의 질문 단위(운영, 설정, 매핑)로 변경

### Task 3. 모드 칩 일관성 강화
- 파일 후보: `web/src/components/layout/global-top-bar.tsx`
- 작업:
  - 모드 칩 너비 최소값 지정으로 레이아웃 점프 방지
  - 칩 텍스트 패턴 통일: `{모드명} 모드`

### Task 4. 쇼핑몰 상세 페이지 공통 헤더 도입
- 파일 후보:
  - `web/src/app/(app)/settings/shopping/channels/page.tsx`
  - `web/src/app/(app)/settings/shopping/mappings/page.tsx`
  - `web/src/app/(app)/settings/shopping/dashboard/page.tsx`
  - `web/src/app/(app)/settings/shopping/factors/page.tsx`
  - `web/src/app/(app)/settings/shopping/sync-jobs/page.tsx`
- 작업:
  - 페이지별 목적/현재 상태/다음 액션을 동일한 헤더 블록으로 통일

## 5) 검증 체크리스트
- 홈(`/dashboard`) 진입 시:
  - 좌측에 쇼핑몰 그룹이 보이지 않는다.
  - 상단에 `홈 모드`가 보인다.
- 쇼핑몰(`/settings/shopping`) 진입 시:
  - 좌측에 쇼핑몰 전용 그룹만 보인다.
  - 상단에 `쇼핑몰 모드`가 보인다.
- 분석(`/analysis/overview`) 진입 시:
  - 분석 전용 그룹이 보인다.
  - 상단에 `분석 모드`가 보인다.
- 모드 순환 버튼 클릭 시:
  - 홈 -> 쇼핑몰 -> 분석 -> 홈 순서가 유지된다.

## 6) 완료 기준(Definition of Done)
- 사용자 테스트 5회에서 "현재 위치를 바로 알 수 있다" 응답 80% 이상
- 쇼핑몰 핵심 작업(채널 점검, 매핑 확인, 로그 확인) 이동 횟수 평균 20% 이상 감소
- 모드 전환/네비 노출 관련 회귀 이슈 0건

## 7) 롤아웃 순서
1. 네비/모드 라벨 안정화(완료)
2. 쇼핑몰 홈 IA 재구성
3. 상세 페이지 공통 헤더 적용
4. 사용성 점검 후 라벨/순서 미세조정
